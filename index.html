    Mi guía personal para
    crear documentación¶

    En este cuaderno voy a hacer un recorrido por las herramientas que utilizo para reunir mis notas. Mi enfoque no deja de ser uno más entre las miles de soluciones que existen para este propósito. Un simple procesador de textos del tipo Microsoft Word podría bastarnos para crear un repositorio de documentos, pero mi idea va en otra dirección:

    • Me gusta tener los contenidos accesibles desde cualquier ordenador, sin necesidad de una distribución manual. Un servidor web me permitirá alojarlos. Yo subo los textos a GitHub, y los presento con GitHub Pages

    • Organizo la documentación en forma de cuadernos. Cada capítulo de un cuaderno se mostrará en forma de página web. Los enlaces que hay en cada página me permiten saltar de un capítulo a otro.

    • Cada página comenzará siendo un archivo de texto ubicado en mi ordenador personal. Convierto esos archivos a formato web usando un software llamado Material for MkDocs.

    • Para crear los documentos necesitaremos un editor de textos. Hay tres tipos de archivos:

      • Archivos binarios, que además del texto, incluyen imágenes, información de tipo de letra, formatos de página, etcétera. Es algo típico de editores del tipo Microsoft Word.

      • Archivos de "solo texto" (plain text), que excluyen todo aquello que no sea una simple secuencia de caracteres.

      • Archivos de texto con formato. Son aquellos que, siendo plain text, respetan unas reglas de escritura e incluyen información en forma de anotaciones insertadas en el texto, popularmente conocidas como "marcas".

    • Yo utilizo el tercer formato, el de archivo plain text con marcas, por lo que he de optar por una sintaxis de entre las muchas que hay disponibles. Redacto los cuadernos en formato markdown, una de las opciones más populares.

    • Para crear documentos markdown nos vale cualquier editor plain text. Algunos de ellos incluyen funcionalidades orientadas a este formato, como por ejemplo, la división de la pantalla en dos paneles, uno de edición y otro de previsualización. En mi caso, uso Visual Studio Code.

    Finalmente, hay que mencionar que Material for MkDocs se ejecuta desde el terminal de comandos, por lo que es importante tener algunos conocimientos de los comandos de nuestro sistema operativo.
