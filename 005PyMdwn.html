    Extensiones PyMdown¶

    PyMdown es una colección de extensiones que amplía las capacidades de Python Markdown. Las utiliza Material for MkDocs para proporcionar una experiencia enriquecida a partir de las funcionalidades estándar de MkDocs.

    Details¶

    Details permite insertar admoniciones en las que solo se muestre la cabecera. Pulsando sobre ella, se muestre el resto de la nota. En lugar de los caracteres !!! usaremos ???.

    ??? question "¿puede ver esta nota correctamente?"

    Si.

    Muestra:

    ¿puede ver esta nota correctamente?

    Si.

    Si queremos el efecto contrario, que la admonición se muestre desplegada y se pueda plegar, sustituir ??? por ???+

    Esta extensión se activa incluyendo en mkdocs.yml:

    markdown_extensions:
  - pymdownx.details

    Emoji¶

    La extensión Emoji permite, como su nombre indica, añadir emojis. Cada uno se escribe indicando su nombre entre caracteres :. Por ejemplo:

    Vámonos de fiesta :tada:

    Muestra:

    Vámonos de fiesta

    Activarlo via mkdocs.yml:

    markdown_extensions:
  - pymdownx.emoji

    Tradicionalmente, se usa la biblioteca de [EmojiOne], pero por motivos de licencias, se recomienda cambiar a otras bibliotecas, por ejemplo twemoji:

    markdown_extensions:
  - pymdownx.emoji
        emoji_index: !!python/name:material.extensions.emoji.twemoji

    La documentación de Material for MkDocs tiene una función de búsqueda de etiquetas

    La opción emoji_generator establece como vamos a generar el emoji:

    markdown_extensions:
  - pymdownx.emoji
        emoji_index: !!python/name:material.extensions.emoji.twemoji 
        emoji_generator: !!python/name:material.extensions.emoji.to_svg

    Mark¶

    La extensión mark permite marcar fragmentos de texto. Deben delimitarse entre dos signos = y otros dos de cierre:

    Texto normal, ==y este otro marcado==.

    Muestra:

    Texto normal, y este otro marcado.

    Se activa en mkdocs.yml con:

    markdown_extensions:
    - pymdownx.mark

    Tilde¶

    La extensión tilde proporciona la posibilidad de mostrar texto rayado. Debemos delimitarlo con dos símbolos ~ a cada lado:

    ~~texto suprimido~~

    Se activa en mkdocs.yml con:

    markdown_extensions:
    - pymdownx.tilde

    Caret¶

    La extensiones Caret permite subrayar texto, delimitándolo entre dos parejas ^^: 

    ^^Texto subrayado^^

    Muestra:

    Texto subrayado

    También permite crear superíndices, delimitándolos entre caracteres ^:

    H^2^O

    Muestra:

    H2O

    Activarla en mkdocs.yml:

    markdown_extensions:
  - pymdownx.caret

    Smart symbols¶

    La extensión SmartSymbols convierte determinadas secuencias de texto en caracteres especiales. Por ejemplo:

    Markdown Resultado
    (tm) ™
    (c) ©
    (r) ®
    c/o ℅
    +/- ±
    --> →
    <-- ←
    <--> ↔
    =/= ≠
    1/4, etc. ¼, etc.
    1st 2nd etc. 1st 2nd etc.

    Se activa con:

    markdown_extensions:
    - pymdownx.smartsymbols

    Arithmatex¶

    La extensión Arithmatex genera y muestra ecuaciones matemáticas. Por ejemplo, si escribimos:

    $$
\cos x=\sum_{k=0}^{\infty}\frac{(-1)^k}{(2k)!}x^{2k}
$$

    Obtendremos

    \[ \cos x=\sum_{k=0}^{\infty}\frac{(-1)^k}{(2k)!}x^{2k} \]

    Activar esta extensión con:

    markdown_extensions:
   - pymdownx.arithmatex:
         generic: true

    Arithmatex utiliza un software llamado MathJax. Consultar la documentación para conocer la sintaxis a usar en la escritura de fórmulas.

    Además de activar la extensión, en el archivo de configuración [mkdocs.yml] debemos incluir:

    extra_javascript:
  - javascripts/mathjax.js
  - https://unpkg.com/mathjax@3/es5/tex-mml-chtml.js

    Y creamos un archivo docs/javascripts/mathjax.jscon los siguientes scripts:

    window.MathJax = {
  tex: {
    inlineMath: [["\\(", "\\)"]],
    displayMath: [["\\[", "\\]"]],
    processEscapes: true,
    processEnvironments: true
  },
  options: {
    ignoreHtmlClass: ".*|",
    processHtmlClass: "arithmatex"
  }
};

document$.subscribe(() => { 
  MathJax.startup.output.clearCache()
  MathJax.typesetClear()
  MathJax.texReset()
  MathJax.typesetPromise()
})

    BetterEm¶

    La extensión BetterEm mejora las reglas de detección de asteriscos y caracteres de subrayado para mostrar negrita y cursiva. La activamos en mkdocs.yml:

    markdown_extensions:
  - pymdownx.betterem

    BetterEm requiere que no haya espacios entre los marcadores y el texto:

    Esto *se mostrará en cursiva*
Esto * no tendrá efecto *

    Se permiten elementos anidados

    ***Cursiva y nerita* solo negrita.**

***Negrita y cursiva** solo cursiva.*

    Ver Referencia

    Caption¶

    La extensión Caption permite poner un título a cualquier bloque Markdown, como es el caso de imágenes, tablas, y bloques de texto fuente de programa. Activarlo en mkdocs.yml:

    markdown_extensions:
  - pymdownx.blocks.caption

    El título se pone tras el bloque en cuestión:

    Fruta      | Cantidad
---------- | ------
Manzanas   | 20
Plátanos   | 3
Melón      | 1

/// caption
Existencias en almacén
///

    Muestra:

    Critic¶

    La extensión Critic utiliza un marcado especial para añadir mostrar los cambios que se han hecho en el texto, el texto eliminado, lo insertado y las sustituciones, permitiendo además añadir comentarios.

    Para indicar un texto añadido, delimitarlo entre {++ y ++}. Para indicar un texto suprinmido, usar {-- y --} en su lugar. Por ejemplo:

    Se mostrará {--el texto que se ha eliminado--}{++el texto nuevo++}.

    Activamos esta extensión en mkdocs.yml:

    markdown_extensions:
  - pymdownx.critic

    La opción mode indica como se visualizará el texto final:

    markdown_extensions:
  - pymdownx.critic:
      mode: view

    Por defecto, el valor es view, lo que visualiza ambos texto , marcando con colores el texto antiguo tachado, y el nuevo subrayado.

    El modo accept muestra el texto nuevo, mientras que el modo reject muestra la versión antigua.

    Cuando se trata de indicar una sustitución, podemos delimitar todo el conjunto entre ~~ y escribir el texto eliminado seguido del nuevo, separados por ~>:

    Se mostrará {~~el texto que se ha eliminado~>el texto nuevo~~}.

    También podemos resaltar texto específico con {==texto a marcar==}. O incluso añadir un comentario con {>>texto del comentario<<}. O por ejemplo, para insertar un bloque de comentarios:

    {>>
texto línea 1
texto línea 2
texto línea 3
<<}

    Bloques de código delimitados¶

    En el estándar markdown, un bloque sangrado con cuatro espacios será considerado como texto de un programa (código fuente), mostrado con un tipo de letra especial, y respetados los saltos de línea y espaciado extra. Por ejemplo:

    Texto de un programa 
en lenguaje C:

    /* Programa para mostrar la palabra Hola */

    #include <stdio.h>

    int main(void)
    {
        puts("Hola");
        return 0;
    }

    Muestra:

    Texto de un programa en lenguaje C: 
    
/* Programa para mostrar la palabra Hola */
#include 

int main(void)
{
    puts("Hola");
    return 0;
}

    Véase que las dos primeras líneas, que forman parte de un párrafo regular, se unen en un único párrafo, pero el bloque sangrado respeta los saltos de línea, el espaciado entre palabras, se muestra en un tipo de letra especial, y se suprime los cuatro espacios de sangría.

    MkDocs incluye la extensión Fenced code blocks, que proporciona una alternativa a los bloques de código sangrados. En lugar de sangrar el texto, el bloque a respetar comienza por una línea formada por tres o más caracteres ~ o bien acentos invertidos. Por ejemplo:

    Texto de un programa 
en lenguaje C:

~~~~~~~~~~~~~~~~~~~~~
/* Programa para mostrar la palabra Hola */

#include <stdio.h>

int main(void)
{
    puts("Hola");
    return 0;
}
~~~~~~~~~~~~~~~~~~~~~

    o bien

    ``````````````````
texto a respetar
``````````````````

    Estas líneas delimitadoras no se mostrarán en el resultado final.

    Pero hay más. Si se trata del texto de un programa, podremos resaltar sus elementos en diferentes colores. Tenemos que especificar el lenguaje de resaltado:

    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~ .c
texto del programa
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

    El punto es opcional. Podemos usar llaves y añadir un nombre identificador a todo el bloque:

    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~ {.c #ejemplo-1}
texto del programa
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

    Para que el resaltado de código fuente funcione, necesitamos activar la extensión CodeHilite en el archivo de configuración:

    markdown_extensions:
    - CodeHilite

    Para que esta extensión funcione, tenemos que tener instalada la biblioteca pygments de python. En principio se instala con el comando:

    pip3 install pygments

    Pero la instalación de Material for MkDocs resuelve esto automáticamente. Comprobamos que [pygments] está instalada con el comando:

    pip3 list

    Véase la lista de lenguajes contemplados en esta página.

    En el caso de bloques identificados al estilo tradicional, mediante sangrado, también podemos identificar el lenguaje en una primera línea.Por ejemplo, si queremos mostrar un ejemplo de programación en lenguaje Python:

    Texto regular.

    :::python
    texto del bloque a resaltar

    Podemos mostrar a la izquierda una barra con números de línea. Esto hay que configurarlo en mkdocs.yml:

    markdown_extensions:
  - codehilite:
      linenums: true

    Podemos resaltar determinadas líneas añadiendo el argumento hl_lines tras el nombre del lenguaje. Las líneas se cuentan a partir de 1:

    ``` python hl_lines="3 5"
texto del bloque resaltado
```

    Pero todo lo visto tiene una limitación. Los bloques delimitados no pueden sangrarse, por lo que no pueden colocarse dentro de otros elementos: listas, citas, etc. Material for MkDocs soluciona esto prescindiendo de las extensiones Fenced code blocks y CodeHilite, y utilizando en su lugar Superfences e Highlight.

    SuperFences¶

    Esta extensión sustituye a Fenced code blocks, y permite sangrar un bloque delimitado. Por ejemplo, el texto a mostrar en una admonición ha de sangrarse:

    !!! Note "Ejemplo de programa"

    ```
    texto del programa a mostrar
    ```

    Esta extensión ha de ser activada en el archivo mkdocs.yml:

    markdown_extensions:
  - pymdownx.superfences

    Highlight¶

    La extensión Highlight añade soporte para el resaltado de sintaxis en bloques delimitados con SuperFences. La activamos en mkdocs.yml:

    markdown_extensions:
  - pymdownx.highlight
  - pymdownx.superfences

    Se admiten varias opciones:

    markdown_extensions:
  - pymdownx.highlight:
      use_pygments: true
  - pymdownx.superfences

    use_pygments establece si vamos a utilizar la biblioteca de resaltado pygments, o un programa JavaScript en su lugar. En el primero de los casos, podemos añadir otras opciones:

    markdown_extensions:
  - pymdownx.highlight:
      pygments_lang_class: true
      auto_title: true
      linenums: true
      linenums_style: pymdownx-inline
      anchor_linenums: true
      line_spans: __span
    • pygments_lang_class añade una clase CSS para identificar el lenguaje utilizado.
    • auto_title añade un título a cada bloque, indicando el lenguaje utilizado.
    • linenums añade números de línea.
    • linenums_style selecciona uno de los métodos disponibles para generar números de línea:
      • table pone los números en una celda de tabla
      • pymdownx-inline genera los números como parte de la línea
    • anchor_linenums genera un enlace a cada número
    • line_spans encierra cada línea entre etiquetas <span>

    InlineHilite¶

    La extensión InlineHilite proporciona resaltado para fragmentos de código fuente dentro de un párrafo regular. Requiere activar highlight:

    markdown_extensions:
  - pymdownx.highlight
  - pymdownx.inlinehilite

    Estos pequeños fragmentos de código se delimitan entre apóstrofos. Tras el primer apóstrofo tenemos que escribir el lenguaje, precedido por tres símbolos :::.

    ```markdown
Usar las etiquetas `:::html <p> ... <p>` para delimitar párrafos.
```

    Keys¶

    La extensión keys permite generar imágenes de teclas. Por ejemplo, para mostrar Ctrl+Alt+Del escribimos:

    ++Ctrl+Alt+Del++

    Véase referencia de teclas. Algunos ejemplos:

    Markdown Muestra
    ++space++ Space
    ++backslash++ \
    ++brace-left++ {
    ++colon++ :
    ++comma++ ,
    ++double-quote++ "
    ++equal++ =
    ++arrow-up++ Up
    ++arrow-down++ Down
    ++arrow-left++ Left
    ++arrow-right++ Right

    Activarlo en mkdocs.yml:

    markdown_extensions:
  - pymdownx.keys

    Snippets¶

    La extensión [snippets] permite incluir texto de otros ficheros en el actual. Por ejemplo:

    Texto regular

```

```

Más texto

    El nombre del fichero ha de ir entrecomillado y precedido por la marca --8<--.

    Se activa mediante:

    markdown_extensions:
  - pymdownx.snippets

    Tabbed¶

    tabbed

    La extensión [tabbed] permite agrupar contenidos en pestañas seleccionables. Por ejemplo:

    === "Bloque 1"
    texto **regular**.

    Otro párrafo.

=== "Bloque 2"
    Una lista de elementos.

    - elemento 1
    - elemento 2

    Muestra:

    texto regular.

    Otro párrafo.

    Una lista de elementos.

    • elemento 1
    • elemento 2

    Se activa con:

    markdown_extensions:
  - pymdownx.tabbed:
      alternate_style: true

    La opción alternate_style proporciona una mejor visualización en dispositivos móbiles.

    Tasklist¶

    La extensión tasklist muestra una lista de elementos con viñetas marcadas:

    -   [X] elemento 1
    *   [X] elemento A
    *   [ ] elemento B

        texto añadido

        +   [x] elemento a
        +   [ ] elemento b
        +   [x] elemento c

    *   [X] elemento C
-   [ ] elemento 2
-   [ ] elemento 3

    Muestra:

    • elemento 1

      • elemento A

      • elemento B

        texto añadido

        • elemento a
        • elemento b
        • elemento c

      • elemento C

      • elemento 2
      • elemento 3

    Activarla con:

    markdown_extensions:
  - pymdownx.tasklist:
      custom_checkbox: true
      clickable_checkbox: true

    custom_checkbox=true proporciona un estilo de viñeta mejorado. clickable_checkbox permite cambiar el estado de as viñetas. Indicar con [x] las que se muestran marcadas por defecto.
