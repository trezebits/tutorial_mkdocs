    Markdown¶

    Markdown es una sintaxis sencilla y fácil de editar, ideada en 2004 por John Gruber. Se trata de un proyecto que ha evolucionado muy poco desde su creación, lo que ha dado lugar a que surjan diferentes versiones mejoradas. McDocs, el generador que estoy usando para los cuadernos de notas, utiliza la versión Python Markdown.

    Es posible que el editor que utilicemos, que en mi caso es Visual Studio Code, cuente con un panel de previsualización que utiliza su propia versión de markdown. Esto hace que el aspecto final de las páginas web generadas por MkDocs no coincidirá con la previsualización.

    Podemos ver las especificaciones originales en la página web de John Gruber, daringfirewall. Los propios archivos originales de esta documentación están escritos en formato markdown, y resultan interesantes como ejemplo que podemos estudiar. Se encuentran disponibles en:

    • página main
    • página basics
    • página syntax

    En este capítulo nos limitamos a estas especificaciones básicas.

    Párrafos¶

    Los navegadores web descargan e interpretan archivos de texto en formato HTML. Las líneas de texto se concatenan en una única línea, que se ajustará dependiendo del ancho de la ventana, y los espacios entre palabras se reducen a un espacio.

    En formato markdown, podemos separar los párrafos insertando una línea en blanco entre ellos. Por ejemplo:

    Este es el primer párrafo. Es lo suficientemente
largo como para ocupar varias líneas, que serán visualizadas
como un bloque de texto, sin separación entre líneas.

Este es el segundo párrafo. Dejamos una línea
en blanco para separarlo del primero.

    Esto mostrará:

    Este es el primer párrafo. Es lo suficientemente largo como para ocupar varias líneas, que serán visualizadas como un bloque de texto, sin separación entre líneas.

    Este es el segundo párrafo. Dejamos una línea en blanco para separarlo del primero.

    El formato markdon considera línea en blanco aquella que no contiene caracteres o solo tiene espacios en blanco. Si escribimos varias líneas en blanco entre dos párrafos, el exceso será ignorado y contarán como una sola.

    En los archivos markdown, puede ser una buena idea escribir los textos de forma que en cada línea pongamos una frase, limitando el tamaño de las líneas y mejorando la legibilidad, en la seguridad de que en la conversión se unirán los textos para formar el párrafo completo.

    Saltos de línea¶

    Podemos forzar un salto de línea insertando manualmente una etiqueta html <br/> en mitad del texto. MkDocs respeta las etiquetas html y las deja como parte del texto generado.

    Por ejemplo:

    Este es el primer párrafo. <br> Es lo suficientemente
largo como para ocupar varias líneas, que serán visualizadas
como un bloque de texto, sin separación entre líneas.

Este es el segundo párrafo. Dejamos una línea
en blanco para separarlo del primero.

    Esto mostrará:

    Este es el primer párrafo.
    Es lo suficientemente largo como para ocupar varias líneas, que serán visualizadas como un bloque de texto, sin separación entre líneas.

    Este es el segundo párrafo. Dejamos una línea en blanco para separarlo del primero.

    La etiqueta <br/> es la abreviatura de "break".

    Líneas de cabecera¶

    Nótese que, en estos cuadernos, cada página tiene un párrafo de título y otros de cabecera de sección, que se muestran con un tipo de letra especial. Estos párrafos se denominan cabeceras.

    En fotmato markdown identificamos estos párrafos anteponiendo uno o más caracteres #. Por ejemplo:

    # Título del documento

Texto del documento

    El carácter (o caracteres) # se suprimirá del resultado final. Debemos dejar un espacio en blanco entre la marca # y el texto del título para que tenga efecto.

    Podemos crear una estructura formada por un título general del documento, títulos a nivel de sección (nivel 2), a nivel de subsección (nivel 3) y así sucesivamente hasta 6 niveles. Por ejemplo:

    # Título del documento

## Sección 1

### Subsección 1.1

Párrafo de texto regular

### Subsección 1.2

Otro párrafo de texto regular

    El nivel se determina de acuerdo al número de caracteres #.

    Opcionalmente podemos "cerrar" las líneas de título con más caracteres # a la derecha, que en el resultado final serán suprimidos. Es algo puramente "estético" y ni siquiera hace falta que el número de caracteres # coincida con los colocados a la izquierda:

    ## Título de la sección ######

    Si usamos estos párrafos de cabecera, debemos estructurar el documento de la forma adecuada. Téngase en cuenta que las cabeceras se utilizan con muchos propósitos. Sirven, por ejemplo, para para que MkDocs inserte de forma automática una tabla de contenidos, para crear enlaces a una sección o incluso las utilizan los buscadores de Internet. Es un error usar párrafos de cabecera solo para utilizar un tipo de letra especial. Para ese propósito hay otros mecanismos.

    La sintaxis de títulos basada en prefijos # es conocida como "atx", y fue inventada por Aaron Swartz, uno de los principales colaboradores en el diseño de markdown. Alternativamente existe la posibilidad de utilizar otra forma de sintaxis inspirada en el antiguo formato "Setext". Se trata de una opción que solo admite dos niveles de cabecera. Los títulos se delimitan subrayando con caracteres = las cabeceras de nivel uno, y con guiones las de nivel dos. Por ejemplo:

    Título del documento
====================

Título de la sección
--------------------

Párrafo de texto regular
    El número de caracteres de subrayado no tiene por que coincidir con la longitud del título.

    Sangrado de párrafos¶

    Llamamos párrafo sangrado a aquel en el que todas sus líneas tienen espacios en blanco a la izquierda. Por ejemplo:

    Primer párrafo estándar.

    Este es un párrafo con sangría.
    Tiene 4 espacios a la izquierda.

    Markdown utiliza la sangría con propósitos especiales, y por lo tanto, debemos evitar todo tipo de espaciado a la izquierda excepto para esos casos particulares. El uso más habitual es crear párrafos preformateados.

    Pçárrafos preformateados¶

    Supongamos que estamos creando un manual de programación en lenguaje java. Escribimos algo así como:

    Programa de ejemplo:

class HelloWorld {
          public static void main (String args[]) {
               System.out.println("Hola Mundo");
          }
     }

    Pero al visualizarlo, se unen todas las líneas de texto y el resultado es:

    Programa de ejemplo: class HelloWorld { public static void main (String args[]) { System.out.println("Hola Mundo"); } }

    Queremos que el texto del programa forme un párrafo donde se respeten los saltos de línea y los espacio extra. La solución es marcar todo el texto como párrafo preformateado. Se logra añadiendo un sangrado de cuatro espacios:

    Programa de ejemplo:

    class HelloWorld {
              public static void main (String args[]) {
                   System.out.println("Hola Mundo");
              }
         }

    Y ahora se respeta el espaciado, y se muestra en un estilo de letra especial:

    Programa de ejemplo: 
    
class HelloWorld {
          public static void main (String args[]) {
               System.out.println("Hola Mundo");
          }
     }

    Nótese que un sangrado de tres espacios no tiene efecto. Ha de ser de cuatro. Los espacios extra se mostrarán como tales.

    En los fragmentos preformateados se respetan los espacios extra y no tiene efecto el formato markdown, considerando las marcas como texto literal:

    Texto regular

    # Párrafo de cabecera sangrado

    Muestra:

    Texto regular 
    
# Párrafo de cabecera sangrado

    Citas¶

    Los párrafos de citas se muestran en un formato especial, y markdown las delimita anteponiendo en cada línea un carácter >

    > Esta es una cita con dos párrafos.
> Las líneas de un párrafo se concatenan
> en una sola.
> 
> Este es el segundo párrafo de la cita.

    Muestra:

    Esta es una cita con dos párrafos. Las líneas de un párrafo se concatenan en una sola.

    Este es el segundo párrafo de la cita.

    El tipo de letra y otros elementos de formato dependen del estilo utilizado en nuestra página web, algo que se determina en un archivo CSS de "hoja de estilos", que es generado por MkDocs. Podemos complementarlo con estilos a nuestro gusto.

    El prefijo > solo es necesario en la primera línea de cada párrafo:

    > Lorem ipsum dolor sit amet,
consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

> Donec sit amet nisl. Aliquam semper ipsum sit amet velit.
Suspendisse id sem consectetuer libero luctus adipiscing.

    Aunque es más elegante escribir:

    > Lorem ipsum dolor sit amet,
  consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus.
  Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

> Donec sit amet nisl. Aliquam semper ipsum sit amet velit.
  Suspendisse id sem consectetuer libero luctus adipiscing.

    Lo que muestra:

    Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aliquam hendrerit mi posuere lectus. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

    Donec sit amet nisl. Aliquam semper ipsum sit amet velit. Suspendisse id sem consectetuer libero luctus adipiscing.

    Las citas pueden "anidarse" (una dentro de otra) si añadimos caracteres ">" adicionales:

    > Primer nivel de texto.
>
> > Cita dentro de otra.
>
> Volviendo al primer nivel.

    Lo que muestra:

    Primer nivel de texto.

    Cita dentro de otra.

    Volviendo al primer nivel.

    Un párrafo de cita puede contener otros elementos markdown, incluyendo listas, cabeceras, bloques de código, etc.

    > ## Cabecera.
> 
> 1. Primer elemento.
> 2. Segundo elemento.
> 
> Texto fuente de un programa delimitado mediante sangrado:
> 
>     return shell_exec("echo $input | $markdown_script");

    Listas¶

    Para crear una lista con viñetas podemos usar como marcadores unos asteriscos (*), guiones (-) o símbolos de suma (+). Por ejemplo:

    Lista de colores:

+ rojo
+ verde
+ azul

    El navegador web muestra los elementos de la lista en línea aparte y precedidos por una viñeta:

    Lista de colores:

    • rojo
    • verde
    • azul

    Las listas numeradas se escriben de la siguiente forma

    Lista de colores:

1.  rojo
2.  verde
3.  azul

    Lo que muestra:

    Lista de colores:

    1. rojo
    2. verde
    3. azul

    Sin embargo, la numeración será ignorada en el resultado final. En principio se asume que empieza por 1, ya que, al convertir a formato html, no se indica nada:

    <p>Lista de colores:</p>

<ol>

<li> rojo</li>
<li> verde</li>
<li> azul</li>

</ol>

    La etiqueta <ol> significa ordered list. Es tarea del navegador numerar los párrafos al visualizarlos. En formato markdown, podríamos escribir:

    Lista de colores:

1.  rojo
1.  verde
1.  azul

    ... y el resultado final mostraría números 1, 2, 3 ... ¿Como establecer otra numeración? Insertando de forma manual etiquetas HTML en el texto markdown:

    Lista de colores:

<ol start="9">

<li> rojo</li>
<li> verde</li>
<li> azul</li>

</ol>

    Muestra:

    Lista de colores:

    1. rojo
    2. verde
    3. azul

    Las capacidades del formato HTML van mucho más allá de lo que nos permite markdown, pero este resulta mucho más cómodo, y en el 99% de los textos, su uso es más que suficiente.

    Las listas se pueden sangrar hasta tres espacios (con cuatro sería texto preformateado):

    Párrafo de texto regular.

   - elemento 1
   - elemento 2
   - elemento 3

    Entre el marcador (viñeta/número) y el texto debemos dejar al menos un espacio.

    Un elemento finaliza al introducir el siguiente elemento con un marcador, o bien, al encontrar una línea en blanco. Sucesivas líneas de texto formarán parte del elemento y su unirán en una sola:

    *   Lorem ipsum dolor sit amet,
consectetuer adipiscing elit.
Aliquam hendrerit mi posuere lectus.
Vestibulum enim wisi,  viverra nec, 
fringilla in, laoreet vitae, risus.

*   Donec sit amet nisl.
Aliquam semper ipsum sit amet velit.
Suspendisse id sem consectetuer
libero luctus adipiscing.

    Pero es más elegante escribir el texto alineado:

    *   Lorem ipsum dolor sit amet,
    consectetuer adipiscing elit.
    Aliquam hendrerit mi posuere lectus.
    Vestibulum enim wisi,  viverra nec,
    fringilla in, laoreet vitae, risus.

*   Donec sit amet nisl.
    Aliquam semper ipsum sit amet velit.
    Suspendisse id sem consectetuer
    libero luctus adipiscing.

    Si queremos que un nuevo párrafo forme parte del elemento anterior, debemos alinear el sangrado de la primera línea del párrafo, que debe consistir en al menos cuatro espacios:

    1.  Elemento con dos párrafos.
    Lorem ipsum dolor sit amet,
    consectetuer adipiscing elit.
    Aliquam hendrerit mi posuere lectus.

    Vestibulum enim wisi, viverra nec,
    fringilla in, laoreet vitae, risus.
    Donec sit amet nisl.
    Aliquam semper ipsum sit amet velit.

2.  Suspendisse id sem consectetuer
    libero luctus adipiscing.

    Si colocamos una cita dentro de un elemento, el carácter > debe estar sangrado:

    *   un elemento con cita:

    > Una cita dentro de un
    > elemento de lista

    Para colocar un bloque de texto preformateado dentro de un elemento de lista, tenemos que sangrarlo con al menos ocho espacios:

    *   Un elemento de lista:

        texto preformateado
    Nótese que podríamos crear un elemento de lista accidentalmente. Por ejemplo:

    1986. Un año muy especial.

    La solución es poner una barra invertida \ antes del punto, que no será mostrada. Estas barras hacen que el caracter que viene a continuación se considere como texto regular.

    1986\. Un año muy especial.

    Podemos crear listas anidadas (una dentro de otra) si añadimos sangrado a la lista interna:

    - Elemento 1
    - elemento 1.1
    - elemento 1.2
- Elemento 2

    lo que muestra:

    • Elemento 1
      • elemento 1.1
      • elemento 1.2
    • Elemento 2

    Línea horizontal¶

    Podemos insertar una línea horizontal insertando una etiqueta <hr/> o bien si escribimos en una línea aparte tres o más guiones, asteriscos o caracteres de subrayado. Podemos añadir espacios entre estos símbolos. Por ejemplo:

    * * *

***

*****

- - -

--------------------------

    El aspecto de la línea depende de la hoja de estilos utilizada por MkDocs, o de las que añadamos nosotros.

    Enlaces¶

    En el texto de una página web solemos encontrar enlaces, que al pulsar sobre ellos, nos llevan a otra ubicación. Todo enlace está formado por dos componentes principales:

    • el texto a mostrar
    • la dirección enlazada (URL)

    En formato markdown hay dos formas de especificar un enlace, inline y por referencia. En el formato inline, escribimos el texto del enlace [entre corchetes], seguido de la URL entre paréntesis:

    [texto del enlace](http://ejemplo.com/)

    Todo ello en mitad de un párrafo, si es nuestro deseo:

    Visite [mi página web](http://ejemplo.com/) para más información

    En formato HTML, los enlaces pueden tener opcionalmente un atributo "title", que es un texto que se suele mostrar al pasar el ratón sobre el enlace. Se indica entre comillas a continuación de la URL:

    [texto del enlace](http://ejemplo.com/ "Pulse aquí")

    Si queremos enlazar con un archivo en el mismo repositorio, podemos usar "rutas relativas" a la ubicación del archivo actual:

    Véase [Acerca de](./acercade.html)

    Podemos indicar el archivo markdown en lugar del archivo html generado. MkDocs transformará la URL para sustituirla por el archivo HTML:

    Véase [Acerca de](./acercade.md)

    Nótese que si clonamos un repositorio y llevamos los archivos a otra ubicación, las direcciones relativas funcionarán mejor que las absolutas, ya que no tenemos que ajustar el texto con la nueva ruta.

    La otra forma de crear enlaces en formato markdown es "por referencia". Se sustituye la URL entre paréntesis por una referencia entre corchetes:

    Este es [un ejemplo][1] de enlace por referencia.
Este es [otro ejemplo][2] de enlace por referencia.

    Opcionalmente podemos poner un espacio entre ambos elementos:

    Este es [un ejemplo] [1] de enlace por referencia.

    En algún lugar del documento debemos definir la referencia. Se escribe en línea aparte:

    [1]: http://ejemplo.com/ "Título opcional"
[2]: http://ejemplo2.com/ "Título opcional"

    El sangrado no puede superar los tres espacios. Entre el carácter : y la URL debemos colocar al menos un espacio.

    El atributo title es opcional, y puede escribirse entre comillas, apóstrofos o paréntesis: 

    [id]: http://ejemplo.com/  "Título opcional"
[id]: http://ejemplo.com/  'Título opcional'
[id]: http://ejemplo.com/  (Título opcional)
    La URL puede escribirse entre símbolos < >

    [id]: <http://ejemplo.com/>  "Título opcional"

    Podemos poner el título en la siguiente línea y sangrarlo con espaciado extra si lo deseamos:

    [id]: <http://ejemplo.com/>
      "Título opcional"

    Nótese que estas líneas con definiciones de enlaces no serán visibles en el resultado final.

    Las etiquetas usadas para definir enlaces son case-insensitive. Por ejemplo:

    [texto 1][a]
[texto 2][A]

    se considera que la referencia es la misma.

    Es posible usar el texto del enlace como etiqueta, lo que se conoce como implicit link name. Por ejemplo:

    [Google][]

    O bien, sin la segunda pareja de corchetes:

    [Google]

    Y definimos el enlace como:

    [Google]: http://google.com/

    Esto también funciona cuando el texto contiene espacios:

    Visite [Daring Fireball][] para más información.

    Y definimos el enlace como:

    [Daring Fireball]: http://daringfireball.net/

    Las URLs se pueden colocar en cualquier lugar del documento. Hay quien prefiere ponerlas tras el párrafo en cuestión, y otros prefieren poner todas las definiciones agrupadas al final del documento.

    ¿Para que sirven los enlaces por referencia? Cuando en un documento hay varios enlaces a la misma página, evitan tener que escribir la URL varias veces. Basta con poner el [texto del enlace] allí donde queremos uno, y poner la lista de URLs en algún lugar del documento. Hay quien prefiere dejarlas para el inicio o final del texto, y quien decide poner las URLs tras cada párrafo.

    Enlaces automáticos¶

    Podemos escribir una URL en mitad del texto, que será mostrada como enlace si la escribimos entre delimitadores < >.

    Página web: <http://ejemplo.com/>

    También podemos hacer esto con direcciones de correo:

    Enviar un correo a: <direccion@ejemplo.com>

    Texto enfatizado¶

    Markdown considera los asteriscos (*) y caracteres de subrayado (_) como indicadores de énfasis. Todo texto entre asteriscos o caracteres (_) será mostrado con un tipo de letra enfatizado (generalmente cursiva), y si usamos doble asterisco o subrayado, en negrita.

    Lo siguiente

    *asterisco*
_subrayado_
**doble asterisco**
__doble subrayado__

    Se mostrará como:

    asterisco
    subrayado
    doble asterisco
    doble subrayado

    Podemos poner énfasis en mitad de una palabra:

    Don*super*increíble

    pero todo marcador rodeado de espacios será tratado como texto regular. Si queremos forzar que se trate como tal, podemos "escapar" las marcas precediéndolas con una barra invertida:

    \*texto que se mostrará entre asteriscos\*

    Podemos combinar dos estilos diferentes de enfatizado:

    *Esto es __super__ importante*
    lo que muestra:

    Esto es super importante

    Código fuente inline¶

    Podemos indicar que un fragmento de texto se muestre de forma literal delimitándolo con acentos invertidos (`). A diferencia de los bloques preformateados, un fragmento literal puede formar parte de un párrafo regular. Por ejemplo:

    Usar `*texto entre asteriscos*` para enfatizar

    generará:

    Usar *texto entre asteriscos* para enfatizar

    Para incluir un acento invertido en el texto literal, usar como delimitadores dos acentos:

    ``Usar un carácter (`) delimitador.``

    Los delimitadores pueden escribirse con espacios, uno tras la apertura y otro antes del cierre, que serán eliminados del resultado final.

    Caracteres como < > no tendrán la consideración de etiquetas HTML, sino que se considerarán como texto regular

    No usar etiquetas `</br>`.

    Se muestra como:

    No usar etiquetas </br>.

    Sin los acentos invertidos, se convertiría en un salto de línea.

    Imágenes¶

    La sintaxis para insertar imágenes es la misma que para los enlaces, añadiendo un prefijo !. El texto del enlace será considerado como texto alternativo cuando no se puede mostrar la imagen.

    Las imágenes Inline se escriben así:

    ![texto](/carpeta/img.jpg)

![texto](/carpeta/img.jpg "Título")

    Y las imágenes por referencia:

    ![texto][id]

    la definición es idéntica a la de los enlaces:

    [id]: carpeta/imagen  "Título"

    Si queremos especificar las dimensiones, usar etiquetas html <img> en lugar del formato markdown.

    <img src="imagen.jpg" alt="texto alternativo" height="42" width="42">

    Insertar html¶

    El problema de markdown es que no cubre todas las funcionalidades que proporcionan las etiquetas html, que es una sintaxis de una gran complejidad.

    La solución a aplicar es simple; basta con insertar directamente etiquetas html en los textos markdown cuando sea necesario. En general serán respetadas, salvo cuando entren en conflicto con las etiquetas generadas en el proceso de conversión.

    En líneas generales, las etiquetas utilizadas para delimitar bloques de texto html, tales como <div>, <table>, <pre>, <p> , etc. , deben separarse del resto del documento con una línea en blanco al principio y final del bloque, y las etiquetas de inicio y fin de bloque no deben llevar sangrado alguno.

    Por ejemplo, las funcionalidades proporcionadas por markdown no incluyen la creación de tablas. Podemos insertar una usando directamente la sintaxis html:

    Párrafo de texto regular markdown

<table>
        <tr>
            <td>texto de celda</td>
        </tr>
</table>

Otro párrafo de texto regular markdown

    Nótese que dentro de un bloque html no tienen efecto las marcas utilizadas en la sintaxis markdown. Por ejemplo, un asterisco se visualizará como tal, no como delimitador de texto en cursiva.

    Además de utilizar etiquetas html para delimitar bloques enteros, podemos usar otras etiquetas dentro de un párrafo regular. Por ejemplo, ya hemos visto como utilizar <br> para forzar un salto de línea, o <span>...</span> para aplicar un estilo especial a un fragmento de texto.

    Podemos incluso sustituir el formato markdown por etiquetas html si ese es nuestro deseo. Por ejemplo, para insertar una imagen escribimos:

    ![texto alternativo](imagen.jpg)

    pero podemos hacerlo en formato html:

    <img src="imagen.jpg" alt="texto alternativo" >

    No conviene abusar de esta posibilidad, porque hace los documentos más complejos de leer y estamos renunciando a los beneficios de la sintaxis markdown. Sin embargo, resulta útil si queremos especificar algo de mayor complejidad.

    Escapes¶

    Todo carácter precedido por una barra invertida será tratado como texto regular. Por ejemplo, 

    \*texto con asteriscos\*

    Mostrará los asteriscos como parte del texto, sin que se consideren como delimitadores de letra enfatizada.

    Un carácter \ se considerará como texto regular siempre que no preceda a alguno de los siguientes:

    \   barra invertida
`   acento invertido
*   asterisco
_   subrayado
{}  llaves
[]  corchetes
()  paréntesis
#   hash
+   signo de suma
-   guión
.   punto
!   exclamación

    Enlaces internos¶

    La sintaxis estándar de enlaces es la siguiente:

    [texto](URL)

    MkDocs permite utilizar nombres de documentos markdown como URLs. Al hacer la conversión, sustituirá esa referencia por la página definitiva. Por este motivo debemos utilizar paths relativos:

    Véase [licencia](licencia.md) del proyecto para más detalles.

    En este caso se asume que el documento enlazado está en la misma carpeta que el documento actual. Si el archivo destino está en otro directorio, asegurarnos de incluirlo en el path relativo:

    Véase [licencia](../acercade/licencia.md) del proyecto para más detalles.

    Pero hay una situación en la que esto no funciona. Markdown permite incluir texto HTML "en bruto" en nuestros documentos. Esto implica que los enlaces escritos en ese formato deben incluir una URL en formato html. No es posible usar referencias a ficheros con extensión .md y pretender que las convierta al fichero final .html:

    <a href="otroDocumento.html">texto</a>

    Para incluir imágenes, utilizar la sintaxis markdown habitual:

    ![imagen](img/pantalla.png)

    Si activamos en el archivo de configuración la extensión "toc", que se utiliza para crear "tablas de contenido", podemos beneficiarnos de los IDs generados para cada línea de cabecera en el documento. Añadiendo un carácter # entre el nombre del documento y el de la sección podemos enlazar a secciones concretas:

    Véase [licencia](acercade.md#licencia) del proyecto para más detalles.

    Los IDs se crean deduciéndolos del texto de la cabecera, convirtiéndolo a minúsculas y sustituyendo caracteres no permitidos (incluyendo espacios) por guiones. Varios guiones consecutivos se eliminarán para dejar uno solo.

    Enlaces a secciones¶

    Podemos insertar un enlace a la línea de cabecera de una sección, si escribimos entre corchetes su texto. Por ejemplo:

    ## Acerca de este documento

... texto regular

    Y en algún otro lugar:

    Véase [acerca-de-este-documento]

    MkDocs genera un identificador para cada elemento <h1> ... <h6> que consiste en el texto de la cabecera, en minúsculas, suprimiendo los espacios en blanco y sustituyéndolos por guiones.
